## What Each File Does

### Core Application Files

**src/index.ts** — 60 lines
- Express app setup
- Middleware registration (JSON, trace ID, logging)
- Route mounting (all 8 modules)
- Health check endpoint
- 404 handler
- Error handler
- Server startup (port 4000)

**src/auth/routes.ts** — 120 lines
- POST /auth/signup — Create business + user account
- POST /auth/login — JWT token generation
- Bcrypt password hashing
- Duplicate email checking

**src/businesses/routes.ts** — 60 lines
- GET /businesses/:businessId — Retrieve profile
- PUT /businesses/:businessId — Update profile (ownership verified)

**src/submissions/routes.ts** — 470 lines (LARGEST FILE)
- POST /submissions — Full pipeline:
  - Call MS2 Scout Agent (classify)
  - Hazard flag check (STOP if hazardous)
  - Confidence check (ask followup or STOP if low)
  - Call MS2 Alchemist Agent (match)
  - Confidence floor check (suppress if < 0.7)
  - Create match row + deal event
  - Call MS2 Negotiator Agent (draft)
  - Create outreach drafts (2 rows: source + target)
  - Send SES notification (stub)
- GET /submissions/:submissionId — Retrieve details
- Full error handling + logging

**src/outreach/routes.ts** — 170 lines
- POST /outreach/:id/accept — Accept proposal (ownership verified)
  - Updates draft status
  - Checks if both sides accepted
  - Updates match status if both accepted
  - Logs deal event
- POST /outreach/:id/reject — Reject proposal (ownership verified)
  - Marks entire match as rejected
  - Logs deal event
- GET /outreach/:id — Retrieve draft

**src/verification/routes.ts** — 150 lines
- POST /verification/:matchId/submit — Submit evidence (ownership verified)
- POST /verification/:matchId/confirm — Confirm evidence (business or admin)
- GET /verification/:matchId — Retrieve records

**src/certificates/routes.ts** — 130 lines
- POST /certificates/:matchId/issue — Issue certificate
  - Verifies both verifications confirmed
  - Calls MS2 Verify Agent
  - Creates certificate row
  - Updates match status
  - Logs deal event
- GET /certificates/:certificateId — Retrieve certificate

**src/matches/routes.ts** — 90 lines
- GET /matches/business/:businessId — All matches for a business (ownership verified)
- GET /matches/:matchId — Match details + drafts + events
- GET /matches/:matchId/events — Deal events only

**src/admin/routes.ts** — 120 lines
- GET /admin/queue/verifications — Unconfirmed verifications
- GET /admin/monitoring/low-confidence — Matches < 0.75 confidence
- GET /admin/monitoring/events — Deal event log
- GET /admin/haulers — List haulers
- POST /admin/haulers — Create hauler
- GET /admin/audit/log — Full audit trail
- All endpoints require `req.userRole === 'admin'`
